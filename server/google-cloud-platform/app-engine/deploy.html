Deploy a Dart Application on App Engine

To deploy your Dart application on App Engine Managed VMs, you need to create a Google Cloud project with a unique name. The project name provides part of the URL for your application.

  <my-project-name>.appspot.com

As you follow these instructions, replace <my-project-name> with the name of your project.

Deploy the application

  1. If you have not already done so, put dart/dart-sdk/bin in your PATH environment variable. You will need this for the next step.

  2. Change to the directory that contains the Client/Server Example you used previously and run pub build.

    $ cd appengine_samples/clientserver
$ pub build
$ gcloud preview app deploy app.yaml

    Note that the deploy command can take awhile.

    Note: If you are using custom datastore indices, you also need to specify index.yaml:

    $ gcloud preview app deploy app.yaml index.yaml

  3. Visit the project URL to see the Dart application running.

    http://<my-project-name>.appspot.com/

Congratulations! Youâ€™ve deployed your Dart app on the web using App Engine Managed VMs.

How deployment works

Before deploying, you run pub build on the web directory where the client code lives before building the container. This creates a build directory, which contains all the transformed assets. The application gets assets from here using the same code that is used to get assets from pub serve when developing.
